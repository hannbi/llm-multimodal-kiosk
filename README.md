#  🤖 LLM-Multimodal-Kiosk
LLM 기반 대화형 주문 처리와 음성(STT/TTS)·터치를 결합한 멀티모달 스마트 키오스크

## 💡 Description | 프로젝트 소개 
본 프로젝트는 카페 환경을 가정해 설계한 LLM 기반 멀티모달 주문 키오스크 시스템이다. 기존 키오스크는 화면 중심의 복잡한 메뉴 구조와 다단계 옵션 선택으로 인해 주문 시간이 길어지고, 고령층·비숙련 사용자에게 높은 사용 장벽을 제공해 왔다. 특히 시각장애인이나 터치 조작에 제약이 있는 사용자에게는 화면 기반 입력 방식 자체가 접근성의 한계로 작용한다.

이러한 문제를 해결하기 위해 본 시스템은 사용자의 발화를 자연어 그대로 이해하고 주문으로 연결하는 대화형 주문 방식을 도입하였다. LLM이 사용자의 질의를 문맥 단위로 해석해 주문 의도(Intent)와 세부 정보(Slot)를 추출하며 음성(STT/TTS)과 터치 UI를 결합한 멀티모달 인터페이스를 통해 상황에 따라 유연한 입력 전환이 가능하도록 구성하였다. 사용자는 메뉴 탐색 없이도 “달지 않은 음료로 바꿔줘”, “카페인 적은 메뉴 추천해줘”와 같은 자연스러운 요구만으로 주문을 진행할 수 있다.

또한 영양정보 기반 스마트 추천 기능을 통해 사용자의 취향과 건강 조건을 반영한 개인화된 메뉴 선택을 지원한다. 주문 과정 중 수량 변경, 옵션 수정, 메뉴 삭제 등도 실시간으로 처리되어 주문 오류를 줄이고 전체 주문 흐름의 안정성과 효율성을 높였다. 본 프로젝트는 사용자가 시스템의 구조에 적응해야 했던 기존 키오스크와 달리 **시스템이 사용자의 언어와 맥락을 이해해 동작하는 지능형 키오스크 모델**을 제시한다.

## 📱 Features | 주요 기능
### 1. LLM 기반 주문 처리
- 사용자 발화를 자연어로 이해해 주문 의도(Intent)를 분류하고 메뉴·사이즈·온도·옵션·영양 조건을 Slot 구조로 추출
- “아메리카노 아이스로 바꿔줘”, “카페인 적은 메뉴 추천해줘”와 같은 자연스러운 표현만으로도 대화 문맥을 유지한 주문 처리로 옵션 변경 등 추가 요청을 자연스럽게 지원
### 2. STT·TTS 기반 음성 인터랙션
- 음성 입력을 파일 형태로 서버에 전송한 뒤 Whisper 기반 STT를 통해 텍스트로 변환
- 주문 처리 결과 및 시스템 안내를 Google TTS 기반 음성 출력으로 제공해 화면 조작 없이도 주문 진행을 지원
- 발화 중단, 재요청, 옵션 수정 등 실제 주문 환경을 고려한 음성 인터랙션 흐름을 구성하고, 처리 결과를 UI에 즉시 반영
### 3. 터치·음성 멀티모달 UI/UX
- 터치와 음성 입력을 동시에 지원하는 멀티모달 키오스크 UI를 구축해 상황에 따라 입력 방식을 유연하게 전환할 수 있도록 설계
- 음성으로 주문을 시작한 뒤 화면에서 결제를 진행하거나, 화면에서 메뉴를 선택한 뒤 음성으로 옵션을 수정하는 등 실제 사용 흐름을 반영한 인터랙션을 지원
- 고령층·비숙련 사용자를 고려해 버튼 크기, 색 대비, 옵션 표시 방식을 개선하고 시각장애인을 위한 음성 주문 기본 흐름을 제공
### 4. 영양정보 기반 스마트 추천
- 음료의 칼로리, 당류, 나트륨, 카페인 등 영양 성분 정보를 DB로 구축하고 사용자 조건에 따라 필터링하는 추천 로직을 구현
- “당 적은”, “카페인 없는"과 같은 발화를 인식해 조건에 부합하는 메뉴를 자동 추출
- 추천 결과를 LLM이 자연어 형태로 안내해 기존 키오스크에서 제공하기 어려웠던 개인화된 메뉴 선택을 지원
### 5. 실시간 주문 수정 및 관리
- FastAPI 기반 서버와 주문 상태 저장 구조를 통해 수량 변경, 옵션 수정, 메뉴 삭제 등 주문 내역을 실시간으로 관리
- 자연어 기반 주문 수정·취소를 즉시 반영해 주문 오류를 줄이고 전체 주문 흐름의 안정성과 유연성을 향상
  
## ⚙️ Stack & Structure | 기술스택 & 구조
- Frontend: React  
- Backend: FastAPI  
- Database: SQLite  
- LLM: OpenAI API  
  -> 사용자 발화를 주문 의도(Intent)와 세부 정보(Slot)로 분해하는 구조 설계  
  -> 주문 흐름 제어를 위한 프롬프트 및 응답 포맷(JSON) 커스터마이징  
- Speech  
  -> Whisper STT: 음성 파일 기반 주문 인식 및 주요 주문 처리  
  -> Google Speech-to-Text: 실시간 음성 입력 실험 및 확장 구조  
  -> Google Text-to-Speech: 주문 결과 및 시스템 안내 음성 출력

본 시스템은 React 기반 클라이언트와 FastAPI 서버 간 통신 구조로 구성되며 사용자의 입력(터치·음성)은 주문 상태(state)와 UI에 즉시 반영되도록 설계하였다. 음성 주문은 업로드된 음성 파일을 Whisper STT로 텍스트 변환한 뒤, LLM이 Intent·Slot을 추출하고 결과를 Google TTS로 음성 안내하며 UI 흐름까지 함께 제어한다(STT → LLM → TTS/UI). 

또한 실시간 음성 인식을 위한 Google Speech-to-Text(Streaming) 모듈도 별도로 구성하여, 향후 실시간 주문 환경으로 확장 가능한 구조를 확보하였다.

## 📌 Expected Outcomes | 기대효과
#### - 접근성 및 사용자 경험 향상
본 프로젝트를 통해 기존 키오스크가 갖고 있던 입력 방식 중심의 사용 한계를 개선하고 사용자 중심의 대화형 주문 환경을 제시할 수 있다. 음성(STT/TTS)과 터치를 결합한 멀티모달 인터페이스를 통해 화면 조작에 익숙하지 않은 고령층·비숙련 사용자·시각장애인도 보다 직관적으로 주문에 참여할 수 있어 키오스크 접근성이 크게 향상된다.

#### - 주문 정확도 및 운영 효율 개선
또한 LLM 기반 주문 처리 구조를 통해 사용자의 자연어 발화를 문맥 단위로 이해하고 주문 수정·옵션 변경·추천 요청 등을 실시간으로 반영함으로써 기존 키오스크에서 빈번히 발생하던 주문 오류와 반복 조작을 줄일 수 있다. 이는 주문 정확도 향상과 대기 시간 감소로 이어져 매장 운영 효율 개선에도 기여한다.

#### - 서비스 확장 가능성
영양정보 기반 스마트 추천 기능은 개인의 취향과 건강 조건을 고려한 메뉴 선택을 가능하게 하여 단순 주문 기계를 넘어 개인화된 서비스 플랫폼으로서의 키오스크 확장 가능성을 보여준다. 더 나아가 본 시스템은 카페 환경뿐 아니라 패스트푸드점, 무인 매장, 공공 안내 시스템 등 다양한 무인 서비스 환경으로 확장 적용할 수 있는 지능형 키오스크 모델의 기반을 제시한다.
